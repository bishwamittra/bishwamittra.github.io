Incremental learning of interpretable classification rules

Paper Code Cite Slides



Interpretable machine learning focuses on designing decision boundaries that are easily understood by human. Rule-based classifiers are arguably one of the interpratable models where the decision boundary is represented in the form of rules, for example, decision trees, decision lists etc. In the following part, we are going to talk about decision rules that follow a well known structure in formal logic in computer science.


In the avobe example we show Iris dataset, where each row represents a sample and each column represents a feature/attribute. An Iris flower has four features: sepal length, sepal width, petal lenght, and petal width. In supervised learning, each sample has a class label, for example, there are three classes in Iris dataset: Setosa, Versicolor, and Virginica. We can learn following interpretable rule on Iris dataset.

A sample is predicted Iris Versicolor if
(sepal length $ > 6.3$ $\textbf{OR}$ sepal width $ > 3$ $\textbf{OR}$ petal width $\le 1.5 $ ) $ \textbf{AND} $
(sepal width $ \le 2.7$ $\textbf{OR} $ petal length $ > 4$ $ \textbf{OR} $ petal width $ >1.2 $) $ \textbf{AND} $
(petal length $ \le 5 $)


To compare interpretable rules with black box models, here we also show a neural network, which has hierarchy of neurons and lots of parameters. Therefore, it is often difficult to understand how the model is predictiing in case of black box models. On the other hand, the avobe rule can be parsed easily. The rule comprises of few boolean variables, i.e. sepal length $ > 6.3$ can be either true or false for a sample. An Iris flower is predicted to be Iris Versicolor if it satisfies all three clauses, where a clause is satisfied when at least one literal in that clause is satisfied.
